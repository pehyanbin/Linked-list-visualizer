Why linked lists?

Arrays store elements contiguously, so insertion and deletion often require shifting items.

Average insertion/deletion in an array takes O(n) time because roughly n/2 elements move.

🔀

Singly linked list operations

Each node stores data and one next pointer. The list ends at Nil, represented in C++ as nullptr or a sentinel object.

Iterator state transitions

A doubly linked-list iterator can move like a small deterministic finite automaton: BEFORE, DATA, and AFTER. ++ follows next; -- follows previous.
